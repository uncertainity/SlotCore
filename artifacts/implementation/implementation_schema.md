## Function Contracts
### runOneSpin
Purpose: Resolve one paid base spin and any selected feature branch.
Inputs: wager WagerConfig [1], feature_choice FeatureChoice|null [1], rng RngSource [1]
Outputs: spin_result SpinResult [1]
Pseudocode:
- validate wager against known minimum bet; do not enforce missing maximum or denomination
- select base reelset using B1/B2 weights
- generate base pay window
- apply optional base COR injection
- evaluate base ways, scatter count, and collect wins
- if base scatter count >= 3, require feature_choice and execute selected feature award
- aggregate win buckets and return SpinResult

### selectBaseReelset
Purpose: Select B1 or B2 for a base spin.
Inputs: rng RngSource [1]
Outputs: reelset_name string [1]
Pseudocode:
- weighted choice over B1=58 and B2=42

### generatePayWindow
Purpose: Build a visible window from a named reelset and reel stop positions.
Inputs: reelset_name string [1], rng_or_forced_stops RngSource|int[6] [6]
Outputs: pay_window PayWindow [4x6], stops int[6] [6]
Pseudocode:
- for each reel choose or accept a stop index in [0, reel_size-1]
- treat stop index as top visible symbol
- fill rows downward with (stop_index + row) mod reel_length
- store row-major as pay_window[row][reel]

### injectAdditionalCOR
Purpose: Optionally add extra COR symbols to the base-game window.
Inputs: pay_window PayWindow [4x6], rng RngSource [1]
Outputs: augmented_window PayWindow [4x6]
Pseudocode:
- choose trigger yes/no with weights 1/9
- if no, return unchanged window
- if yes, choose added COR count by weights
- sample eligible positions excluding existing COLLECT, WILD, or SCATTER
- assign each added COR a value from base COR value weights

### evaluateBaseWindow
Purpose: Evaluate all base-game win buckets after final base window is known.
Inputs: pay_window PayWindow [4x6], total_bet number [1]
Outputs: base_evaluation BaseEvaluation [1]
Pseudocode:
- evaluate ways wins
- count SCATTER symbols
- set direct scatter win to zero per human review when SCATTER wins are not mentioned
- evaluate collect bucket if COLLECT and COR/jackpot-bearing COR are visible
- return line, scatter, COR, and trigger metadata

### evaluateWaysWins
Purpose: Evaluate 4096-way left-to-right standard-symbol wins.
Inputs: pay_window PayWindow [4x6]
Outputs: ways_result WaysResult [1]
Pseudocode:
- for each standard symbol, count matching or WILD symbols on consecutive reels from reel 1
- stop at first reel with zero matches
- if matched kind has a paytable tier, ways = product of per-reel match counts
- add paytable factor times ways to line win factor
- WILD substitutes only for standard symbols

### countScatters
Purpose: Count SCATTER symbols anywhere in a window.
Inputs: pay_window PayWindow [4x6]
Outputs: scatter_count int [1]
Pseudocode:
- count cells equal to SCATTER; WILD never substitutes for SCATTER

### evaluateScatterWin
Purpose: Resolve direct SCATTER payout bucket.
Inputs: scatter_count int [1], mode GameMode [1]
Outputs: scatter_win_factor number [1]
Pseudocode:
- return zero direct SCATTER win when SCATTER wins are not mentioned; keep scatter count for triggers and retriggers

### evaluateCollect
Purpose: Evaluate COLLECT/COR collection in base game or Blitz Spins.
Inputs: pay_window PayWindow [4x6], mode GameMode [1], total_bet number [1]
Outputs: collect_result CollectResult [1]
Pseudocode:
- if mode not base_game or blitz_spins return zero
- find COLLECT cells on reel 1 and reel 6
- find all visible COR and jackpot-bearing COR cells
- for each COLLECT independently sum all COR and jackpot values
- multiply collected x-bet sum by total bet for payout amount

### resolveFeatureSelection
Purpose: Map base scatter count and player choice to a feature award.
Inputs: scatter_count int [1], feature_choice FeatureChoice [1]
Outputs: feature_award FeatureAward [1]
Pseudocode:
- require scatter_count in 3..6
- select free_spins or blitz_spins award from table
- do not resolve Buy Bonus because scope is ambiguous

### runFreeSpins
Purpose: Resolve the Free Spins feature.
Inputs: initial_spins int [1], rng RngSource [1], total_bet number [1]
Outputs: feature_result FeatureResult [1]
Pseudocode:
- while remaining spins > 0 generate F1 window
- assign multipliers to visible WILD symbols using weights 2/3/5
- evaluate ways and scatter count
- multiply non-scatter line wins by product of all visible WILD multipliers
- do not multiply scatter bucket
- add retrigger spins for 2 or more SCATTER
- exit when no spins remain

### assignFreeSpinWildMultipliers
Purpose: Assign multipliers to Free Spins WILD symbols.
Inputs: pay_window PayWindow [4x6], rng RngSource [1]
Outputs: wild_multiplier_window number matrix [4x6]
Pseudocode:
- for each WILD cell choose multiplier by weights 2=480, 3=545, 5=360
- non-WILD cells have neutral multiplier 1

### runBlitzSpins
Purpose: Resolve the Blitz Spins feature.
Inputs: initial_spins int [1], rng RngSource [1], total_bet number [1]
Outputs: feature_result FeatureResult [1]
Pseudocode:
- for each awarded spin create an empty 4x6 Blitz window
- place 1 or 2 COLLECT symbols on reel 1 and/or reel 6 with max one per reel
- for every non-COLLECT position resolve COR yes/no
- if zero COR were placed, force one COR
- assign exactly one COR value from Good table and all remaining COR values from Bad table
- evaluate collect
- reset screen for next spin
- no retriggers

### buildBlitzWindow
Purpose: Construct one Blitz Spins window.
Inputs: rng RngSource [1]
Outputs: pay_window PayWindow [4x6]
Pseudocode:
- choose collect count 1 or 2
- place COLLECT only on reel 1 and/or reel 6, max one per reel
- resolve COR on non-COLLECT cells
- force one COR if none appears

### assignCORValues
Purpose: Assign x-bet values to COR symbols for a mode.
Inputs: pay_window PayWindow [4x6], mode GameMode [1], rng RngSource [1]
Outputs: valued_window PayWindow [4x6]
Pseudocode:
- base/injected COR values use base value weights
- Blitz uses Good table exactly once per spin and Bad table for all other CORs
- jackpot-bearing COR values use the same x-bet values as jackpot constants

## Call Graph
- `runOneSpin` -> `selectBaseReelset`, `generatePayWindow`, `injectAdditionalCOR`, `evaluateBaseWindow`, `resolveFeatureSelection`, `runFreeSpins|runBlitzSpins`

- `runBlitzSpins` -> `buildBlitzWindow`, `assignCORValues`, `evaluateCollect`
- `injectAdditionalCOR` -> `assignCORValues`

## Feature Architecture
- Collect: active in base game and Blitz Spins; each COLLECT independently collects all visible COR and jackpot-bearing COR values.
- Free Spins: uses F1, WILD multipliers 2/3/5, retriggers on 2+ SCATTER, exits when spins are exhausted.
- Blitz Spins: creates COR/COLLECT-only screens, guarantees at least one COR and one COLLECT, uses Good table once per spin and Bad table for other CORs, no retrigger.
- Buy Bonus: retained as ambiguous data only; do not implement as player-facing without human review.

## Event Flow
SpinStarted -> ReelsetSelected -> WindowGenerated -> WindowAugmented -> WinEvaluated -> CollectEvaluated -> FeatureTriggered -> FeatureSpinStarted -> FeatureCompleted -> PayoutCalculated
